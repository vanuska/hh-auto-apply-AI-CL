# HH-AUTO-APPLY-AI-CL: Полное руководство по использованию

## Общее описание

Кроссплатформенный инструмент для автоматического поиска вакансий на hh.ru и отправки откликов с персонализированными сопроводительными письмами, сгенерированными искусственным интеллектом. Поддерживает бесплатные модели через OpenRouter, а также платные OpenAI и Anthropic. Включает интерактивный установщик, автоматическое определение операционной системы и полную поддержку WSL, Linux ОС с GUI и поддержку Linux без GUI, через SSH клиента с X11forwarding или через API HH, но `API HH c 2025 года более недоступен для соискателей`.

Внимание! 
Так как бесплатные модели могут не корректно работать с русским или вообще, в моменте, не работать, проеряйте работу AI до боевого запуска, через пункты меню: Шаг 1 (при первом запуске) -> Шаг 2 (при первом запуске, или изменении api или модели) -> Шаг 3 -> или Шаг 4 или Шаг 8.

## Основные возможности

1. Автоматический поиск вакансий через браузер, ограниченно через публичный API hh.ru (см. ниже раздел api.HH.ru) с фильтрацией по ключевым словам, стоп-словам, обязательным словам в названии, режиму работы, наличию тестового задания.
2. Генерация сопроводительных писем с помощью AI (OpenRouter, OpenAI, Anthropic) с адаптацией под конкретную вакансию и профиль кандидата.
3. Интерактивный выбор модели с тестированием на реальном примере письма.
4. Автоотклик через сохраненную браузерную сессию Playwright с поддержкой видимого и headless-режимов.
5. Поддержка серверного Linux без графического интерфейса через xvfb-run.
6. Журналирование всех действий в SQLite базе данных для исключения повторных откликов.
7. Интерактивный установщик setup_tool.py с 10 шагами: установка зависимостей, настройка .env, проверка моделей, создание профиля из PDF/DOCX/TXT или вручную, настройка config.yaml и шаблона письма, авторизация, тест, очистка базы данных, запуск.
8. Извлечение текста из файлов profile: PDF (pypdf), DOCX (python-docx), TXT (любая кодировка), ручной ввод.
9. Поддержка ответов на дополнительные вопросы работодателя (город, зарплатные ожидания и пользовательские ответы по ключевым словам).
10. Автоматическое определение ОС (Windows, Linux Desktop, Linux Server) и установка необходимых зависимостей.

## Структура проекта и файловая система

Корневая директория проекта содержит:

- .env.example - пример файла с переменными окружения
- modules/ - папка с модулями (опционально, для переноса скриптов)

  - profile.md - текстовый профиль кандидата

## Получение API ключей

### api.HH.ru
- При запуске основного скрипта auto_apply.py, без авторизации HH.ru вы получите ответ: HH API search failed: HH API error 403: {"errors":[{"type":"forbidden"}],"request_id":"*************"}
Falling back to hh.ru browser search...
- Для авторизации и доступа к API HH изучите https://api.hh.ru/openapi/redoc. Для соискателей API более не доступен.
- Это не мешает работе скрипта как в скрытом --headless режиме, так и в графическом.

### OpenRouter (рекомендуется, бесплатные модели)

1. Откройте браузер и перейдите на сайт https://openrouter.ai
2. Нажмите кнопку "Sign Up" или "Get Started" в правом верхнем углу
3. Зарегистрируйтесь, используя email и пароль, или войдите через Google/GitHub аккаунт
4. После входа в личный кабинет нажмите на иконку профиля в правом верхнем углу
5. В выпадающем меню выберите "API Keys"
6. Нажмите кнопку "Create New Key"
7. Введите название ключа (например, "hh-auto-apply")
8. Нажмите "Create" и скопируйте полученный ключ (начинается с sk-or-v1-...)
9. Сохраните ключ в надежном месте, он понадобится при настройке .env
10. Для проверки баланса бесплатных кредитов перейдите в раздел "Credits" - OpenRouter дает $1 бесплатных кредитов при регистрации, этого хватит на тысячи запросов к бесплатным моделям

### OpenAI (платные модели)

1. Откройте браузер и перейдите на сайт https://platform.openai.com
2. Нажмите "Sign Up" в правом верхнем углу
3. Зарегистрируйтесь, используя email и пароль, или войдите через Google/Microsoft аккаунт
4. Подтвердите email и номер телефона (требуется для активации аккаунта)
5. Войдите в личный кабинет и перейдите в раздел "API Keys" (находится в меню слева)
6. Нажмите "Create new secret key"
7. Введите название ключа (например, "hh-auto-apply")
8. Выберите разрешения (рекомендуется "All" или "Owner")
9. Нажмите "Create secret key" и скопируйте полученный ключ
10. Пополните баланс в разделе "Billing" (минимальный депозит $5) - без пополнения API не работает
11. Сохраните ключ в надежном месте, он понадобится при настройке .env

### Anthropic (платные модели)

1. Откройте браузер и перейдите на сайт https://console.anthropic.com
2. Нажмите "Sign Up" в правом верхнем углу
3. Зарегистрируйтесь, используя email и пароль, или войдите через Google аккаунт
4. Подтвердите email и номер телефона (требуется для активации аккаунта)
5. Войдите в личный кабинет и перейдите в раздел "API Keys" (находится в меню слева)
6. Нажмите "Create Key"
7. Введите название ключа (например, "hh-auto-apply")
8. Нажмите "Create" и скопируйте полученный ключ (начинается с sk-ant-...)
9. Пополните баланс в разделе "Billing" - без пополнения API не работает
10. Сохраните ключ в надежном месте, он понадобится при настройке .env

### Сравнение провайдеров

- OpenRouter: рекомендуется для начала, есть бесплатные модели, $1 кредитов при регистрации, простой доступ, единый API для всех моделей
- OpenAI: платные модели, требуется пополнение баланса от $5, высокая скорость, широкий выбор моделей (gpt-4o-mini - самая дешевая)
- Anthropic: платные модели, требуется пополнение баланса от $5, высокое качество, модели Claude

## Установка Ubuntu на Windows (WSL)

### Шаг 1: Включение WSL

#Внимание! В WSL команды используют "python", а в нативной Ubuntu "python" или "python3".

1. Откройте меню "Пуск" и введите "PowerShell"
2. Нажмите правой кнопкой мыши на "Windows PowerShell" и выберите "Запуск от имени администратора"
3. В открывшемся окне введите команду: wsl --install
4. Дождитесь завершения установки (может занять несколько минут)
5. После установки перезагрузите компьютер

### Шаг 2: Установка дистрибутива Ubuntu

1. Откройте Microsoft Store (магазин приложений Windows)
2. В поиске введите "Ubuntu"
3. Выберите "Ubuntu 22.04.4 LTS" или "Ubuntu 24.04.4 LTS" (рекомендуется последняя версия)
4. Нажмите "Установить"
5. Дождитесь завершения установки (около 1-2 ГБ)
6. Нажмите "Открыть" после установки

### Шаг 3: Первоначальная настройка Ubuntu

1. При первом запуске Ubuntu предложит создать пользователя и пароль
2. Введите имя пользователя (латиницей, например: user)
3. Введите пароль (не отображается при вводе)
4. Подтвердите пароль
5. Запомните эти данные - они понадобятся для входа в систему

### Шаг 4: Обновление системы

1. В открывшемся окне Ubuntu введите команду: sudo* apt update  *(для windows можно и без sudo, но если нужен, то включить через Система-Дополнительно-Включить sudo)
2. Введите пароль пользователя (созданный на предыдущем шаге)
3. Дождитесь завершения обновления списка пакетов
4. Введите команду: sudo apt upgrade -y
5. Дождитесь завершения обновления пакетов (может занять несколько минут)

### Шаг 5: Установка необходимых пакетов

1. В Ubuntu введите команду: sudo apt install -y python python-pip python-venv git wget curl
2. Дождитесь завершения установки
3. Проверьте версию Python: python --version (должна быть 3.8 или выше)

### Шаг 6: Настройка доступа к проекту из Windows

1. В Windows создайте папку для проекта, например: C:\Users\Ваше_Имя\hh-auto-apply
2. В Ubuntu путь к этой папке будет: /mnt/c/Users/Ваше_Имя/hh-auto-apply
3. Перейдите в эту папку: cd /mnt/c/Users/Ваше_Имя/hh-auto-apply
4. Теперь все файлы будут доступны как из Windows, так и из Ubuntu

### Шаг 7: Запуск в фоновом режиме (опционально)

1. Для работы в фоне используйте screen или tmux
2. Установите screen: sudo apt install -y screen
3. Запустите screen: screen
4. В screen выполните: python auto_apply.py --schedule --apply
5. Отключитесь от screen: Ctrl+A, затем D
6. Для возврата в screen: screen -r

### Доступ к файлам из Windows

- Все файлы проекта доступны в папке C:\Users\Ваше_Имя\hh-auto-apply
- Можно редактировать файлы в любом текстовом редакторе Windows (Notepad++, VS Code)
- В Ubuntu изменения будут видны сразу

### Возможные проблемы и решения

- Ошибка "WSL is not installed": выполните wsl --install в PowerShell от администратора
- Ошибка "Ubuntu not found in Store": проверьте, что вы в Microsoft Store, а не в обычном поиске
- Ошибка "Permission denied": используйте sudo или проверьте права на папку
- Ошибка "python not found": установите Python: sudo* apt install -y python python-pip
- Медленная работа: WSL использует файловую систему Windows, для ускорения перенесите проект в домашнюю папку Ubuntu: ~/hh-auto-apply

### Шаг 10: Запуск auto_apply.py
Предлагает два режима: dry-run (--once, без отправки) и реальный запуск (--once --apply). На Linux-сервере без GUI предлагает выбрать headless-режим или использование xvfb-run. При dry-run записывает в БД статус "dry_run".

## Ручные параметры auto_apply.py

Запуск из командной строки:
python auto_apply.py [--once] [--schedule] [--apply] [--headless] [--max-applications N] [--config PATH] [--state-db PATH]

Параметры:
- --once - однократный запуск (поиск и обработка)
- --schedule - запуск по расписанию из config.yaml (09:30 и 18:30)
- --apply - реальная отправка откликов (без этого флага - dry-run)
- --headless - запуск браузера в headless-режиме (без окон)
- --max-applications N - ограничение числа откликов за запуск
- --config PATH - путь к config.yaml (по умолчанию my/config.yaml)
- --state-db PATH - путь к SQLite базе (по умолчанию data/hh_auto_apply.sqlite3)

Примеры:
# Запуск с отправкой откликов, браузер в фоне
python auto_apply.py --schedule --apply --headless
# Запуск без отправки (только поиск и генерация писем) с видимым браузером
python auto_apply.py --schedule
# Запуск с отправкой, браузер видимый
python auto_apply.py --schedule --apply

## Как зафиксировать модель без интерактивного выбора

В файле .env установите:
OPENROUTER_MODEL=google/gemma-4-31b-it:free
Тогда auto_apply.py будет использовать эту модель всегда, пропуская интерактивный выбор. Если модель не работает, можно снова включить auto: OPENROUTER_MODEL=auto

- Windows: без изменений, браузер открывается в видимом режиме
- Linux с GUI: без изменений, браузер открывается в видимом режиме
- Linux без GUI (сервер): при установке (шаг 1) ставится xvfb, при запуске (шаг 10) предлагает headless или xvfb-run. При ручном запуске можно использовать xvfb-run -a python auto_apply.py --once --apply

## База данных SQLite

Таблица vacancy_runs:
- vacancy_id TEXT PRIMARY KEY - ID вакансии
- status TEXT NOT NULL - статус: success, error, skipped, dry_run
- reason TEXT - причина ошибки или пропуска
- title TEXT NOT NULL - название вакансии
- employer TEXT NOT NULL - работодатель
- url TEXT NOT NULL - ссылка на вакансию
- query TEXT NOT NULL - поисковый запрос
- letter TEXT - сгенерированное письмо
- updated_at TEXT NOT NULL - дата и время обработки

## Управление моделями и API ключами

Все переменные окружения загружаются из .env:
- LLM_PROVIDER: openrouter, openai, anthropic, none

- OPENROUTER_MODEL: модель (auto или конкретная)

- ANTHROPIC_API_KEY: ключ от Anthropic
- ANTHROPIC_MODEL: модель Anthropic

## Особенности работы с резюме

- profile.md имеет приоритет над PDF
- При отсутствии profile.md ищет все PDF в my/ и извлекает текст
- DOCX извлекается через python-docx, DOC требует ручной конвертации (выводится инструкция)
- TXT читается в кодировке UTF-8 с fallback на другие кодировки
- Профиль обрезается до 6000 символов для AI, описание вакансии до 4000 символов

## Логика генерации письма

- Системный промпт: шаблон из cover_letter_prompt.md + HR_ADAPTATION_RULES
- Пользовательский промпт: профиль (6000 символов), вакансия (название, компания, описание 4000 символов), требования к письму
- AI генерирует письмо 3-5 предложений, максимум max_chars (по умолчанию 1200)
- После генерации применяется нормализация: замена ё на е, тире на дефисы, удаление лишних пробелов
- Ответ очищается от английских фраз и объяснений через clean_letter_response()

## Управление расписанием

В config.yaml раздел schedule.run_times:
- Два времени запуска в формате "HH:MM"
- При запуске с --schedule скрипт работает вечно, вычисляя время до следующего запуска
- Каждый запуск выполняет run_once с теми же параметрами (apply/headless)

## Ответы на вопросы работодателя

application_questions в config.yaml:
- city: город (заполняет поля с ключевыми словами "город", "откуда", "проживает")
- salary_expectations: зарплатные ожидания (заполняет поля с ключевыми словами "зарплат", "зп", "ожидания", "доход")
- answers: список с keywords и answer (пользовательские ответы)
При заполнении формы отклика скрипт ищет текстовые поля и input[type='text'], проверяет контекст (окружающий текст) и заполняет подходящие поля.

## Отладка и логирование

- Все действия записываются в БД с временем
- При ошибках отправки создаются скриншоты и HTML-страницы в data/debug/apply-{vacancy_id}-{timestamp}.png и .html
- Статусы файлов показываются в главном меню setup_tool.py
- При запуске auto_apply.py выводятся все найденные вакансии, сгенерированные письма и результаты

## Рекомендации по настройке

1. Сначала выполнить все шаги setup_tool.py по порядку
2. В .env указать рабочий API ключ OpenRouter (бесплатный на openrouter.ai)
3. В config.yaml настроить ключевые слова под свою специализацию
4. В profile.md подробно описать опыт, навыки и достижения (чем подробнее, тем лучше письма)
5. Запустить test_letter.py для проверки качества писем
6. Запустить auto_apply.py --once для dry-run (без отправки) и проверить, что вакансии находятся
7. Если всё устраивает, запустить auto_apply.py --once --apply
8. Для регулярных запусков использовать systemd или cron

## Известные ограничения

- Требуется стабильный интернет и доступ к hh.ru и API выбранного провайдера
- Сайт hh.ru может показывать капчу при интенсивном использовании (рекомендуется delay_between_applications_seconds не менее 10-15 секунд)
- Не все бесплатные модели OpenRouter стабильны, рекомендуется проверять через test_letter.py перед использованием
- Старые .doc файлы не поддерживаются напрямую, требуется конвертация в .docx или .txt
- На серверном Linux для видимого режима требуется xvfb (автоматически устанавливается при шаге 1), но для первичной авторизации требуется GUI и SSH c X11Forwarding

## Полный цикл работы

1. Пользователь запускает setup_tool.py и проходит все шаги
2. Создаются .env, profile.md, config.yaml, cover_letter_prompt.md
3. Выполняется авторизация на hh.ru (сохраняется сессия)
4. При запуске auto_apply.py загружается профиль, конфиг, выбирается модель
5. Осуществляется поиск вакансий через API hh.ru с фильтрацией
6. Для каждой подходящей вакансии генерируется письмо через AI
7. В dry-run письмо сохраняется в БД без отправки
8. В реальном режиме открывается браузер, осуществляется переход на страницу вакансии, заполняется форма отклика, вставляется письмо, заполняются дополнительные вопросы, нажимается кнопка отправки
9. Результат (success/error/skipped) сохраняется в БД
10. По расписанию процесс повторяется автоматически
