## 1. Краткий обзор

Этот инструмент — готовое решение для автоматизации поиска вакансий и отправки персонализированных откликов на HH.ru. Он использует искусственный интеллект (через OpenRouter, OpenAI или Anthropic) для генерации сопроводительных писем, адаптированных под ваше резюме и описание вакансии.

**Главные преимущества:**
- Поддержка бесплатных AI-моделей (OpenRouter)
- Интерактивный выбор модели при первом запуске
- Безопасный режим dry-run без реальных откликов
- Универсальный установщик setup_tool.py
- Работает на Windows, Linux (Desktop и Server)

## 3. Быстрый старт

1. Клонируйте репозиторий:
   git clone https://github.com/vanuska/hh-auto-apply-AI-CL.git
   cd hh-auto-apply-AI-CL

2. Запустите интерактивный установщик и следуйте инструкциям:
   python setup_tool.py

   Установщик проведёт вас через все шаги: установка зависимостей, настройка .env, создание профиля, авторизация на HH.ru, тест письма.

3. После завершения настройки проверьте работу:
   python auto_apply.py --once         # сухой запуск (без отправки)

4. Когда всё устраивает, выполните реальный отклик:
   python auto_apply.py --once --apply

## 4. Установка и настройка

### Автоматическая (рекомендуемая)

Запустите setup_tool.py и выберите шаги по порядку:

┌────┬─────────────────────────────────────────────────────────────────────────┐
│ №  │ Действие                                                              │
├────┼─────────────────────────────────────────────────────────────────────────┤
│ 1  │ Установка зависимостей (библиотеки + Playwright)                      │
│ 2  │ Создание .env (API-ключи, выбор провайдера и модели)                 │
│ 3  │ (Опционально) Проверка доступных моделей                              │
│ 4  │ Создание profile.md (из PDF, DOCX, TXT или вручную)                  │
│ 5  │ Настройка config.yaml (ключевые слова, город, зарплата)              │
│ 6  │ Настройка шаблона письма cover_letter_prompt.md                      │
│ 7  │ Авторизация на HH.ru (сохранение сессии)                              │
│ 8  │ Тест генерации письма                                                 │
│ 9  │ (Опционально) Очистка базы данных                                     │
│ 10 │ Запуск auto_apply.py (выберите dry-run или отправку)                  │
└────┴─────────────────────────────────────────────────────────────────────────┘

### Ручная (если не хотите использовать setup_tool.py)

1. Установите зависимости:
   pip install -r requirements.txt
   playwright install chromium

2. Создайте .env (скопируйте .env.example) и заполните:
   HH_USER_AGENT=hh-auto-apply/1.0 (ваш@email.ru)
   LLM_PROVIDER=openrouter
   OPENROUTER_API_KEY=sk-or-v1-...
   OPENROUTER_MODEL=auto

3. Создайте папку my и поместите туда:
   - profile.md – ваше резюме в формате Markdown
   - config.yaml – конфигурацию (можно взять config.example.yaml)
   - cover_letter_prompt.md – шаблон письма

4. Выполните авторизацию:
   python hh_login.py

5. Проверьте генерацию письма:
   python test_letter.py

### Файл my/config.yaml – основные секции

vacancies:
  keywords: ["Менеджер ИТ", "Руководитель ИТ"]
  required_title_words_any: ["Руководитель", "Менеджер"]
  stop_words: ["стажер", "junior"]

  skip_already_applied: true

  area: 113          # 113 – Россия
  per_page: 20
  max_pages: 1
  title_only: true

  max_applications_per_run: 5
  delay_between_applications_seconds: 12

  city: "Москва"
  salary_expectations: "от 270000 RUB"

  max_chars: 1200
  prompt_path: my/cover_letter_prompt.md

  run_times:
    - "09:30"
    - "18:30"

## 6. Запуск

### Основные команды

┌───────────────────────────────────────────┬──────────────────────────────────────────────────────┐
│ Команда                                  │ Описание                                             │
├───────────────────────────────────────────┼──────────────────────────────────────────────────────┤
│ python auto_apply.py --once              │ Dry-run: только поиск и генерация, без отправки    │
│ python auto_apply.py --once --apply      │ Реальный отклик (отправляет письма)                 │
│ python auto_apply.py --once --apply -…   │ Ограничить количество откликов (например, 2)        │
│ python auto_apply.py --schedule --apply  │ Запуск по расписанию (из config.yaml)               │
│ python auto_apply.py --once --apply -…   │ Headless-режим (без окна браузера)                  │
└───────────────────────────────────────────┴──────────────────────────────────────────────────────┘

### Дополнительные скрипты

- python hh_login.py – авторизация на HH.ru и сохранение сессии
- python test_letter.py – тест генерации письма с текущими настройками
- python check_models.py – показывает все доступные бесплатные модели OpenRouter
- python clean_db.py – очищает базу данных (создаёт бэкап)

## 8. Особенности для Linux-сервера

- setup_tool.py автоматически определяет отсутствие графического интерфейса и предлагает установить xvfb для эмуляции дисплея
- При запуске через setup_tool.py вы можете выбрать режим:
  - headless (рекомендуется) – браузер работает в фоне
  - с xvfb-run – эмуляция дисплея (если сайт блокирует headless)
- Для ручного запуска на сервере используйте:
  xvfb-run -a python auto_apply.py --once --apply
  или флаг --headless
